pyinstaller 打包静态资源

2021-01-08
2021-01-10
pyinstaller, python

Pyinstaller 打包图片等静态资源遇到不少坑，在本机运行一般没什么问题，主要是放到其他电脑运行会出现各种问题，经过不断的测试整理出以下方案，很稳定，换目录换电脑都能正常运行。

如下图，loveu.py 是程序的入口文件，程序里面使用相对路径引用了当前文件夹下三个图片

直接使用命令 pyinstaller -F loveu.py 生成的 exe 文件复制到其他目录，或者放到其他电脑上运行就会提示找不到图片。

解决方法:

第一步

使用 pyi-makespec 命令生成 .spec 文件，这个文件是 pyinstaller 打包时都会生成的

第二步

修改 loveu.spec 文件

修改之前 datas 列表为空

将三个图片分别添加到列表中，每个元素都是个元组，元组内第一个字符串是图片相对于执行程序的路径，因为这几个图片跟 loveu.py 在同一目录下所以只写文件名就行。第二个字符串是程序运行时查找的相对路径，程序运行时会临时生成一个临时目录，在这个临时目录中寻找图片等静态文件，这里设置为点 .，也就是当前目录寻找

注意：程序中引入图片也要使用相对路径，而不是绝对路径，如下面这种方式

修改后

第三步

打包

打包的是上一步设置好的 .spec 文件，而不是最开始的 .py 文件

exe 单文件生成在 dist 目录下，这是一个 pygame 写的小游戏，可以看到图片都正常加载，复制到其他电脑也能正常运行。
